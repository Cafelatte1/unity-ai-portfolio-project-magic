## README 작성 프롬프트 (긍정 중심 / 프로젝트 설명 + 최종 평가 반영)

당신은 GitHub `README.md`를 작성합니다.
아래 자료를 읽고, 이 프로젝트가 **어떤 게임인지**를 명확히 설명하며, `FinalResult`의 코드베이스 평가 내용도 README에 포함하되 **부정적 표현보다 긍정적 성과/강점/개선 방향 중심**으로 자연스럽게 녹여서 작성해 주세요.

### 1) 입력 자료(반드시 읽기)

1. `ProjectDocs/GameOverview.md` (있다면)
2. `CodeEvaluation/FinalResult_YYYY-MM-DD_HH-MM.md` (최종 종합 평가 문서)
3. `Assets/Scripts/` 코드 구조(폴더 트리 수준) 또는 아키텍처 문서(있다면)

> 자료가 없거나 불완전하면, 있는 정보만으로 작성하고 **추측은 최소화**해 주세요. 추측이 필요하면 “추정” 섹션에 짧게 표시합니다.

---

### 2) 작성 원칙(강제)

* 문장 톤은 “자신감 있고 깔끔하게”, 과장 없이 구체적으로 씁니다.
* 평가 내용은 **문제 나열/비판** 대신 아래 방식으로 표현합니다.

  * ✅ 강점/잘된 점을 먼저 제시
  * ➕ “개선 중/다음 단계”를 **전향적 로드맵** 형태로 표현
  * ❌ “위험, 부채, 문제, 부족” 같은 직설적 단어는 가능한 한 피합니다.
* 단, 점수/팩트는 FinalResult를 기준으로 **왜곡 없이** 반영합니다.

---

### 3) 출력 형식(README 마크다운으로 전체 작성)

#### A. 프로젝트 한 줄 소개

* 장르/핵심 재미/차별점이 드러나게 1~2문장

#### B. 게임 소개

* 플레이 목표 / 핵심 루프(예: 탐험→전투→성장 등)
* 주요 특징 4~8개(불릿)
* (있다면) 스크린샷/영상 섹션 (없으면 placeholder)

#### C. 주요 시스템 하이라이트

파일 나열 없이 “시스템 단위”로 정리합니다. 각 항목은 2~4줄.

* Player & Control
* Combat / Skill / Damage
* UI / Chat / UX
* FSM/BT/Event Flow
* AI Inference(OpenVINO 등) 파이프라인
* Tools / Debug / Logging (있다면)

#### D. 기술 스택

* Unity 버전/URP 여부/패키지
* AI/추론(OpenVINO, 네이티브 플러그인 등)
* 플랫폼(PC/Android 등)

#### E. 아키텍처 요약(필수)

* 텍스트 다이어그램 1개 이상(필수)

  * 시스템 관계 / 이벤트 흐름 / 추론 흐름 중 하나 이상 포함
* 설계 포인트 3~6줄: “왜 이렇게 설계했는지”를 명확히

#### F. 코드 품질 요약 (FinalResult 기반, 긍정 중심으로)

`CodeEvaluation/FinalResult_...md`를 기반으로 아래를 포함합니다.

1. **10개 항목 점수 요약 표**

* (1)~(10) 항목명을 짧게 축약해서 표로 작성
* 점수는 `X/5` 형태

2. **Quality Highlights (강점 중심 3~6개)**

* FinalResult의 “Best parts”를 기반으로 재구성
* 가능하면 “구조/확장/성능/안정성/추론 통합” 관점으로 정리

3. **Next Improvements (전향적 로드맵 3~6개)**

* FinalResult의 “즉시 개선/중장기 리팩토링”을

  * “리스크/문제”가 아니라
  * “다음 단계로 더 좋아질 포인트”로 표현
* 예: “입력 처리 안정화 강화”, “이벤트 흐름 표준화”, “추론 리소스 관리 고도화” 같은 톤

> 주의: 이 섹션에서 부정적 단어로 길게 설명하지 말고, “개선 방향 + 기대 효과” 중심으로 짧게 씁니다.

#### G. 실행 방법

* Requirements
* 설치/실행 단계
* (있다면) 데모 씬/테스트 방법

#### H. 폴더 구조(요약)

* 역할 기반 트리 + 각 폴더 책임 1줄

#### I. 로드맵

* 마일스톤 5~10개
* “Next Improvements”와 연결되면 좋음

#### J. 라이선스/크레딧

* 알려진 범위만 기재

---

### 4) 최종 출력

* `README.md` 전체 내용을 완성된 형태로 마크다운 출력해 주세요.
